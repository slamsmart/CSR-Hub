Buatkan aplikasi web full-stack enterprise bernama "CSR Hub" (alternatif nama: "SinergiCSR"), yaitu platform marketplace dua arah yang mempertemukan perusahaan/industri dengan NGO, komunitas, sekolah, koperasi, yayasan, dan startup sosial untuk pengajuan, seleksi, pendanaan, monitoring, dan pelaporan program CSR secara transparan, aman, modern, dan sangat kaya fitur.

Tujuan utama aplikasi:
1. Menjadi pusat kolaborasi CSR nasional yang kredibel.
2. Memudahkan pengusul membuat proposal terstandar dan terverifikasi.
3. Memudahkan perusahaan menemukan program CSR yang relevan berdasarkan wilayah, kategori, anggaran, SDGs, dan dampak.
4. Menyediakan pelacakan, monitoring, audit, dan pelaporan dampak secara real-time.
5. Memiliki keamanan siber tingkat tinggi, siap untuk data sensitif, dokumen legal, dan transaksi pendanaan.

Buat aplikasi yang production-ready, responsif, scalable, modular, dan memiliki UI/UX premium, profesional, modern, elegan, dan mudah dipakai oleh pengguna Indonesia.

TECH STACK
- Frontend: Next.js terbaru + TypeScript + Tailwind CSS + shadcn/ui atau komponen UI modern setara
- Backend: Node.js + Next.js API routes atau NestJS
- Database: PostgreSQL
- ORM: Prisma
- Auth: NextAuth/Auth.js atau sistem auth aman setara
- Storage: cloud/local object storage untuk dokumen proposal, laporan, dan bukti kegiatan
- Realtime: WebSocket atau polling efisien untuk notifikasi dan status
- Charts: library visualisasi modern
- AI Features: rekomendasi matching proposal-perusahaan, ringkasan proposal, klasifikasi kategori, deteksi kelengkapan proposal
- Deployment-ready: konfigurasi environment, struktur project rapi, reusable component, clean code

CYBER SECURITY
Aplikasi ini wajib memiliki keamanan siber yang sangat kuat. Terapkan best practice keamanan sejak desain awal:
1. Authentication & Authorization
- Secure authentication
- Strong password policy
- MFA / 2FA
- Email verification
- Session management aman
- RBAC ketat
- Optional device/session history
- Account lockout / rate limiting untuk brute force

2. Data Protection
- Hash password dengan algoritma kuat
- Enkripsi data sensitif saat transit dan saat tersimpan
- Secure file upload validation
- Malware scanning strategy untuk file upload
- Signed URL / akses file terproteksi
- Kolom sensitif dienkripsi
- Backup strategy

3. App Security
- Validasi input ketat
- Sanitasi output
- Cegah XSS, CSRF, SQL Injection, SSRF, IDOR, file upload abuse, path traversal
- Gunakan secure headers
- CSP policy
- CORS policy yang ketat
- Rate limiting dan throttling
- Proteksi API dari abuse
- Anti-spam untuk form publik

4. Audit & Compliance
- Audit logs detail
- Login history
- Suspicious activity detection
- Risk scoring
- Error logging aman tanpa membocorkan data sensitif
- Retention policy
- Privacy-by-design
- Siapkan fondasi kepatuhan terhadap UU Perlindungan Data Pribadi Indonesia

5. Infrastructure Security
- Environment variables aman
- Secret management
- Prinsip least privilege
- Middleware keamanan
- Monitoring dan alerting
- Dependency security checks
- Secure default configuration

6. Coding Security
- Type-safe code
- Server-side validation
- Zod atau validator setara
- Prisma query aman
- Error handling yang aman
- Jangan hardcode secret
- Jangan expose stack trace ke user
